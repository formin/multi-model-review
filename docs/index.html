multi-model-review

Multi-model code review for Spec-Driven Development.

Build with one LLM (Claude Code, Codex CLI, Gemini CLI, ...), review with a different one. A Claude Code plugin built on top of GitHub Spec Kit.

v0.1.0 MIT Claude · Codex · Gemini

Why multi-model?

A single LLM reviewing its own output tends to rationalize rather than challenge. A different model reading the same spec and diff has:

  • Different training data — catches issues the builder's model is blind to.
  • Different calibration on severity — less confirmation bias.
  • Different house style — flags idiomatic drift the builder normalizes.

Because the handoff is plain markdown on disk, the two models never have to talk to each other — and any LLM with a CLI can play reviewer.

Install

Slash commands (/plugin …) are typed inside a Claude Code session — not at your OS shell. Running them at PowerShell or bash returns '/plugin' is not recognized. Shell commands (claude …, git …) go at your OS shell.

In a Claude Code session:

/plugin marketplace add https://github.com/formin/multi-model-review
/plugin install spec-cross-review@multi-model-review

Or at your OS shell (equivalent):

claude plugin marketplace add https://github.com/formin/multi-model-review
claude plugin install spec-cross-review@multi-model-review
claude plugin list

Then run /reload-plugins (or restart Claude Code) and /multi-model-review will appear in the slash-command list. Default scope is user; add --scope project to restrict to one project.

Update or reinstall

  • Update (in Claude Code): /plugin update spec-cross-review@multi-model-review — or at your shell: claude plugin update spec-cross-review@multi-model-review.
  • Uninstall (in Claude Code): /plugin uninstall spec-cross-review@multi-model-review
  • Full wipe and reinstall: see README — Update / reinstall.
  • Dev-mode (claude --plugin-dir) clone already exists? At the shell, git pull inside it, or delete the directory (Remove-Item -Recurse -Force on PowerShell / rm -rf elsewhere) and re-clone.

Commands

Single entry point with subcommand flags:

InvocationPurpose
/multi-model-reviewDefault — show current status.
/multi-model-review --cross-review [init|status]Init config or show roles, base ref, and packages.
/multi-model-review --review-package [slug] [--base <ref>]Export the handoff bundle for the reviewer model.
/multi-model-review --apply-review [path] [--min-confidence N]Ingest the reviewer's report and drive remediation.

Supported reviewers

Out of the box: Claude, Codex, Gemini. Adding a new reviewer is a one-file change — drop templates/<id>-review-prompt.md and reference the ID in .cross-review/config.json.

Quick workflow

  1. Run /multi-model-review --cross-review init to set builder/reviewer roles.
  2. Build your change (Spec Kit or ad-hoc), then /multi-model-review --review-package to export a handoff bundle.
  3. Run the reviewer model on the bundle in a separate terminal.
  4. Run /multi-model-review --apply-review to ingest findings and walk through fixes.
